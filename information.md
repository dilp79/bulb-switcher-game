# Техническое устройство проекта

## Архитектура

Новая версия проекта больше не использует Phaser-сцены. Вместо этого приложение собрано как статический SPA на HTML, CSS и модульном JavaScript.

Причина смены архитектуры:
- игра не использует физику, тайловые карты или sprite-heavy сцену;
- основная сложность была в UX, хранении данных и редакторе уровней;
- DOM и SVG дают более простую адаптивную раскладку и меньшую связность кода.

## Визуальная система

После UX-polish интерфейс опирается на один стабильный visual language:
- тёмная сцена с сеткой и мягкими световыми пятнами;
- янтарный цвет для primary-action и циановый для системных сигналов;
- `Space Grotesk` для display-иерархии и `Outfit` для основного UI;
- игровое поле и редактор оформлены как единый пульт, а не как набор разрозненных экранов.

## Основные модули

### `js/app/constants.js`

Общие константы проекта:
- лимиты генератора;
- ключи `localStorage`;
- пути к изображениям и звукам;
- базовые подписи UI.

### `js/app/logic.js`

Чистая игровая логика:
- нормализация уровня;
- применение нажатия кнопки;
- проверка победы;
- генерация связей;
- генерация стартового состояния;
- расчёт сложности;
- валидация уровня.

### `js/app/storage.js`

Работа с `localStorage`:
- последний уровень;
- результаты;
- флаг прохождения;
- пользовательские уровни;
- настройки звука.

### `js/app/level-repository.js`

Единый источник данных по уровням:
- читает `levels/manifest.json`;
- загружает все встроенные блоки;
- подмешивает пользовательские уровни;
- отдаёт ссылки на следующий уровень;
- сохраняет и удаляет пользовательские уровни.

### `js/app/audio.js`

Минимальный аудио-слой:
- предзагрузка звуков;
- отключение звука через настройки;
- проигрывание событий UI и победы.

### `js/app/app.js`

Главный orchestrator приложения:
- роутинг между экранами;
- обработка кликов и клавиатуры;
- рендер экранов;
- игровой таймер;
- оверлеи подсказок и победы;
- редактор уровней.

## Данные уровней

### Встроенные уровни

Встроенные уровни по-прежнему лежат в JSON-файлах внутри `levels/block_*`.

Для новой реализации добавлены:
- `levels/manifest.json` — список всех встроенных блоков;
- `levels/block_4/index.json`;
- `levels/block_5/index.json`.

Это убирает старую жёсткую зависимость от трёх блоков и позволяет UI видеть весь каталог из репозитория.

### Пользовательские уровни

Пользовательские уровни сохраняются в `localStorage`, потому что браузерная версия не может писать обратно в файловую систему репозитория.

Legacy-индекс `levels/user_levels/index.json` больше не используется и удалён из проекта, чтобы структура не обещала запись в файловую систему там, где её нет.

## Игровой экран

Игровой экран теперь состоит из:
- шапки с названием уровня и статистикой;
- центральной панели с лампочками и кнопками;
- SVG-оверлея для линий подсказки;
- боковой панели с легендой состояний;
- победного overlay с салютом и переходом на следующий уровень.

## Редактор

Редактор содержит:
- имя уровня;
- stepper-контролы для числа кнопок, лампочек и цветов;
- визуальную схему со связями;
- матрицу связей;
- ручное переключение стартового состояния;
- сохранение, удаление и немедленный запуск уровня.

## Проверка и обслуживание

Для проверки каталога встроенных уровней используется:

```bash
node scripts/validate-levels.mjs
```

Во время полной переделки был найден и исправлен битый файл:
- `levels/block_1/level_3.json` — отсутствовала запятая в массиве `connections`.

Во время чистки каталога были также удалены:
- `assets/images/confetti.png` — неиспользуемый дубликат;
- `levels/user_levels/index.json` — legacy-артефакт старой схемы хранения.
