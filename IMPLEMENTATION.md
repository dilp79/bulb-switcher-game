# Implementation Details

Этот документ нужен для дальнейших доработок проекта.

## Архитектура

Проект больше не использует Phaser. Текущая версия это статический SPA на HTML, CSS и модульном JavaScript.

Почему так:

- в игре нет тяжёлой scene-логики или физики;
- основная сложность здесь не в рендер-движке, а в правилах, данных уровней, редакторе и UX;
- DOM + SVG дают более простую адаптивную раскладку и меньше лишней связности.

## Корневая структура

- `index.html` — entrypoint приложения;
- `css/style.css` — весь визуальный слой;
- `js/main.js` — bootstrap;
- `js/app/` — логика приложения;
- `levels/manifest.json` — каталог встроенных блоков;
- `levels/block_*/*.json` — встроенные уровни;
- `scripts/validate-levels.mjs` — проверка встроенных уровней.

## Модули

### `js/app/constants.js`

Общие константы:

- лимиты генерации;
- ключи `localStorage`;
- пути к ассетам;
- общие подписи UI.

### `js/app/logic.js`

Чистая игровая логика:

- нормализация уровня;
- расчёт сложности;
- применение нажатия кнопки;
- проверка победы;
- генерация связей;
- генерация стартового состояния;
- валидация уровня;
- поиск решения для проверки solvability.

### `js/app/storage.js`

Хранение в `localStorage`:

- последний открытый уровень;
- пройденные уровни;
- лучшие результаты;
- пользовательские уровни;
- настройки звука.

Есть fallback на memory storage, если `localStorage` недоступен.

### `js/app/level-repository.js`

Единый источник истины для уровней:

- читает `levels/manifest.json`;
- загружает все встроенные блоки;
- подмешивает пользовательские уровни из `localStorage`;
- умеет отдавать ссылки на следующий уровень;
- сохраняет и удаляет пользовательские уровни.

### `js/app/audio.js`

Минимальный аудио-слой:

- предзагрузка;
- выключение звука;
- проигрывание событий UI, подсказки и победы.

### `js/app/app.js`

Главный orchestrator:

- роутинг между экранами;
- игровой session state;
- редактор;
- рендер HTML;
- keyboard shortcuts;
- SVG overlay для линий связей;
- victory overlay;
- обновление HUD таймера без полного перерендера приложения каждую секунду.

## Модель уровня

Каждый уровень после нормализации имеет поля:

```json
{
  "id": "optional-for-user-levels",
  "name": "Level name",
  "buttons_count": 3,
  "bulbs_count": 3,
  "colors_count": 2,
  "connections": [[0, 0], [1, 2]],
  "initial_states": [1, 0, 2],
  "target_states": [0, 0, 0],
  "complexity": 2.3,
  "block": "block_1"
}
```

`colors_count` это общее число состояний лампы вместе с `выкл`, а не число дополнительных цветов.

- `2` => `выкл`, `зелёный`
- `3` => `выкл`, `зелёный`, `жёлтый`
- `4` => `выкл`, `зелёный`, `жёлтый`, `красный`

## Игровые правила в коде

- кнопка влияет на все лампы, перечисленные в `connections`;
- каждое состояние переключается по модулю `colors_count`;
- победа это точное совпадение `currentStates` и `target_states`;
- подсказка одноразовая, показывает связи выбранной кнопки на 3 секунды;
- пользовательский уровень не сохраняется, если у него нет решения.

## Каталог уровней

### Встроенные

Встроенные блоки перечислены в `levels/manifest.json`.

Сейчас поддерживаются:

- `block_1`
- `block_2`
- `block_3`
- `block_4`
- `block_5`

### Пользовательские

Пользовательские уровни живут только в `localStorage`.

Старая идея с `levels/user_levels/index.json` удалена. Браузерная версия не пишет в файловую систему, поэтому репозиторий не должен делать вид, что там есть живой каталог для user-generated content.

## Визуальная система

Текущий visual language:

- dark console background;
- primary-action через янтарный свет;
- системные сигналы и secondary-action через циан;
- `Space Grotesk` для display-иерархии;
- `Outfit` для UI и длинного текста;
- поле и редактор стилистически едины.

## Скрипты проверки

### `node scripts/validate-levels.mjs`

Проверяет:

- что все встроенные JSON-файлы читаются;
- что размеры state-массивов корректны;
- что значения `initial_states` и `target_states` укладываются в диапазон `0..colors_count-1`;
- что индексы связей не выходят за границы;
- что уровень проходит базовую валидацию;
- что встроенный уровень реально решаем.
