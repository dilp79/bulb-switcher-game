# Bulb Switcher

Bulb Switcher это браузерная головоломка про скрытые связи между кнопками и лампочками.

Игрок видит состояние ламп, но не видит схему напрямую. Каждое нажатие меняет сразу несколько элементов, и задача в том, чтобы восстановить логику системы и погасить весь щит.

## GDD

### Концепт

- жанр: puzzle / logic;
- платформа: web, статический сайт;
- сессия: короткие уровни с быстрым рестартом;
- основной кайф: понять скрытую проводку по косвенным эффектам.

### Основной игровой цикл

1. Игрок открывает уровень.
2. Нажимает на кнопки и наблюдает, какие лампы меняются.
3. Строит гипотезу о связях.
4. Доводит все лампы до состояния `выключено`.
5. Получает результат по времени и ходам.
6. Переходит к следующему уровню или в редактор.

### Правила

1. У каждой кнопки есть скрытые связи с одной или несколькими лампочками.
2. Нажатие на кнопку переключает все связанные с ней лампы.
3. `colors_count` в данных уровня это общее число состояний лампы, включая `выкл`.
4. Состояния циклические: `выкл -> зелёный -> жёлтый -> красный -> ... -> выкл`.
5. Для текущих встроенных блоков это читается так: `2` состояния это `выкл/зелёный`, `3` состояния это `выкл/зелёный/жёлтый`, `4` состояния это `выкл/зелёный/жёлтый/красный`.
6. Цель уровня это привести все лампочки к состоянию `выкл`.
7. На уровень даётся одна подсказка: она на 3 секунды показывает связи выбранной кнопки.

### Контент

- встроенные блоки уровней из репозитория;
- пользовательские уровни, сохраняемые в `localStorage`;
- редактор уровней с ручной сборкой связей;
- проверка пользовательских уровней на решаемость перед сохранением;
- система лучших результатов по времени и количеству ходов.

### UX-направление

Интерфейс сделан как glowing puzzle console, а не как обычный utilitarian dashboard.

- тёмная сцена с янтарными и циановыми световыми акцентами;
- сильная display-типографика;
- игровое поле это главный визуальный объект;
- редактор живёт в том же визуальном языке, что и сама игра.

## Что есть в проекте

- главный экран с продолжением последнего уровня;
- экран выбора уровней;
- экран игры;

## Запуск

Проект нужно открывать через статический сервер, потому что уровни читаются через `fetch`.

```bash
cd /home/dilp79/apps/2Games/BulbSwitcher
python3 -m http.server 4173
```

Потом открыть:

```text
http://127.0.0.1:4173/
```

Если браузер держит старый кэш после предыдущих версий проекта, сделайте hard refresh: `Ctrl+Shift+R`.

## Полезные команды

Проверка встроенных уровней:

```bash
node scripts/validate-levels.mjs
```

Скрипт проверяет:

- что все встроенные JSON-файлы читаются;
- что `initial_states` и `target_states` остаются в допустимом диапазоне состояний;
- что связи не выходят за границы поля;
- что встроенный уровень реально имеет решение.
