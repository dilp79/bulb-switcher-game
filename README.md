# Bulb Switcher

Статическая браузерная головоломка про скрытые связи между кнопками и лампочками.

## Что сейчас есть

- главный экран с продолжением последнего уровня;
- каталог всех встроенных блоков из репозитория;
- пользовательские уровни в `localStorage`;
- счётчик ходов, таймер, рекорды;
- одна подсказка на уровень с показом реальных связей;
- редактор уровней с генерацией связей и стартового состояния;
- адаптивный интерфейс без Phaser и без фиксированного canvas;
- отдельное визуальное направление в стиле glowing puzzle console.

## Визуальное направление

Текущая версия интерфейса строится вокруг одной идеи: это не просто сайт про игру, а сам игровой пульт.

- тёмная консольная сцена с тёплым янтарным и холодным циановым светом;
- display-типографика для заголовков и более спокойный шрифт для UI;
- игровое поле как главный визуальный объект, а не просто сетка карточек;
- редактор оформлен в том же языке, чтобы не ощущаться отдельным utilitarian-экраном.

## Правила

1. Каждая кнопка меняет состояния связанных лампочек.
2. Состояния циклические: `выкл -> зелёный -> жёлтый -> красный -> ... -> выкл`.
3. Цель — привести все лампочки к состоянию `выкл`.
4. Подсказка доступна один раз за уровень.

## Структура проекта

- `index.html` — entrypoint приложения.
- `css/style.css` — весь визуальный слой.
- `js/app/` — модульная логика приложения.
- `levels/manifest.json` — каталог встроенных блоков.
- `levels/block_*/*.json` — встроенные уровни.
- `scripts/validate-levels.mjs` — проверка встроенного каталога уровней.
- `PRD_ACT.md` — каноническая спецификация проекта.
- `information.md` — техническое описание текущей реализации.

## Локальный запуск

Проект нужно открывать через статический сервер, потому что уровни и манифест читаются через `fetch`.

Пример:

```bash
cd /home/dilp79/apps/2Games/BulbSwitcher
python3 -m http.server 4173
```

После этого откройте:

```text
http://127.0.0.1:4173/
```

## Проверка данных уровней

```bash
node scripts/validate-levels.mjs
```

Скрипт проверяет:
- что все JSON-файлы читаются;
- что индексы связей не выходят за границы;
- что длины `initial_states` и `target_states` совпадают с `bulbs_count`;
- что уровень проходит базовую валидацию.

## Текущие активы

Используются только файлы из `assets/`:
- лампочки `bulb_off`, `bulb_green`, `bulb_yellow`, `bulb_red`;
- кнопка `button`;
- салют `confetti.gif`;
- звуки `button_click`, `hint`, `error`, `victory`.

Новые изображения и звуки без согласования не добавляются.

- удалён legacy-индекс `levels/user_levels/index.json`, потому что пользовательские уровни живут только в `localStorage`.
